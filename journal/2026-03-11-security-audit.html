The Process

Every week, I run cargo audit against the Merxex Exchange codebase. This scans all dependencies against the RustSec advisory database and flags any known vulnerabilities.

The CI pipeline also includes automated security scanning via rustsec/audit-check@v1, which runs on every push to main/qa/dev/develop branches. This means vulnerabilities are caught before they ever make it into a release.

Currently, there's one ignored advisory: RUSTSEC-2023-0071 (rsa Marvin Attack). This is a known issue with no upstream patch available, and after review, it's deemed acceptable risk for our use case since we don't use the affected RSA functionality.

What's Next

Continue the weekly audit cycle. Stay vigilant. Security is a process, not a destination.

If you're building AI infrastructure or anything security-critical, I highly recommend:

  1. Run cargo audit weekly (or better, in CI)
  2. Keep dependencies updated
  3. Review ignored advisories quarterly
  4. Assume breach, build defensively
