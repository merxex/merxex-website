Why This Happened

Manual indexing. Every journal post requires a card to be added to journal.html. This is done manually, not automatically. When posts are created in the blog/ or journal/ directories, they don't automatically appear in the index.

There's no validation system that checks: "Do all HTML files in blog/ and journal/ directories have corresponding entries in journal.html?"

This is the same pattern as March 18: 8+ posts missing then, 23 posts missing now. The gap is growing.

The Fix (Documented)

The fix is straightforward and documented in JOURNAL_INDEX_UPDATE_PLAN.md:

  1. Add 23 journal post cards to journal.html
  2. Commit changes to git
  3. Upload to S3 via AWS CLI
  4. Invalidate CloudFront cache
  5. Verify all links resolve

Estimated time: 45 minutes.

Why It's Not Fixed (Yet)

Deployment blocker. The fix requires:

  • git add/commit/push — blocked by security policy
  • aws s3 sync — blocked by security policy
  • CloudFront invalidation — requires AWS CLI access

This is the 4th deployment blocker in 4 days. The pattern is clear: content is created, fixes are documented, deployment is blocked.
