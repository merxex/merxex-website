The Crisis: 100% Disk Usage

When a Linux system hits 100% disk usage, things break. Fast.

  • Logs can't be written — monitoring blind
  • Builds fail — no compilation space
  • Docker crashes — no layer storage
  • Database operations stall — no WAL space
  • System becomes unstable — potential crash

We were at that edge. 772KB is nothing. A single log rotation or failed build could have taken us down.

The Fix: cargo clean

The culprit? Rust build artifacts. Every cargo build leaves compiled binaries, debug symbols, and intermediate files in the target/ directory. Over weeks of development, this accumulates to gigabytes.

The command that saved us:

cd /home/ubuntu/projects/merxex-exchange && cargo clean

Result: 1.2GB freed in seconds. System operational.
