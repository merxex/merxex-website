The Deployment

On March 7, 2026, the Merxex development environment went live. Here's what was deployed:

48 AWS Resources

  • VPC — Isolated network with public/private subnets
  • ECS Fargate — Serverless container orchestration
  • Aurora Serverless v2 — Auto-scaling PostgreSQL
  • ElastiCache Redis — In-memory caching layer
  • CloudFront — Global CDN with DDoS protection
  • S3 — Static asset storage
  • Application Load Balancer — Multi-AZ traffic distribution
  • ECR — Container registry for Docker images
  • Route53 — DNS management
  • CloudWatch — Metrics, logs, and alarms
  • Security Groups — Network-level access control
  • ACM Certificates — TLS/SSL for all endpoints

Architecture Principles

This infrastructure was built with five non-negotiable principles:

1. Security First

Network isolation: All databases and caches are in private subnets with no internet access. Only the ALB is publicly accessible.

Encryption everywhere: Aurora encryption at rest, TLS in transit, ElastiCache encryption, S3 bucket encryption. No exceptions.

Least privilege: Security groups allow only necessary ports. ECS tasks run with minimal IAM permissions.

2. No Single Points of Failure

Multi-AZ deployment: VPC spans us-east-1a and us-east-1b. If one AZ fails, the other takes over.

Aurora Multi-AZ: Database automatically replicates across availability zones. Failover is automatic and transparent.

CloudFront global edges: Content is cached at 110+ edge locations worldwide. Origin failure doesn't mean downtime.

3. Auto-Scaling from Day One

Aurora Serverless v2: Scales from 0.5 to 128 ACUs based on load. No capacity planning needed.

Fargate auto-scaling: ECS service scales from 1 to 10 tasks based on CPU/memory utilization.

CloudFront caching: Handles traffic spikes without origin ever seeing the load.

4. Observability Built-In

CloudWatch alarms: CPU, memory, health checks, database connections, storage capacity. All monitored, all alerting.

Structured logging: JSON-formatted logs with correlation IDs. Easy to trace requests across services.

Metrics dashboard: Real-time visibility into all critical metrics. No guesswork about system health.

5. Cost-Conscious (But Not Cheap)

Serverless where it makes sense: Aurora Serverless, Fargate, CloudFront. Pay for what you use.

Reserved capacity where it doesn't: ElastiCache is a fixed cost, but it's critical for performance. Not cutting corners here.

Free tier maximized: CloudFront free tier covers first $10/mo of traffic. Lambda free tier covers orchestration. Every dollar counts.

What's Not Here (And Why)

No Kubernetes: Overkill for this stage. Fargate is simpler, cheaper, and scales just as well.

No self-hosted monitoring: Prometheus/Grafana would be a nice-to-have. CloudWatch is good enough for now.

No multi-region: Premature optimization. Single-region Multi-AZ handles 99.99% of failure scenarios.

The Philosophy

Infrastructure should be adequate, not impressive. If it's reliable, secure, and cost-effective, it's good infrastructure. Everything else is noise.

Cost Breakdown (Dev Environment)

Transparency matters. Here's what the dev environment costs per month:

  • ECS Fargate: ~$15/mo (1 task, 0.5 vCPU, 1GB RAM)
  • Aurora Serverless v2: ~$40/mo (minimum 0.5 ACU)
  • ElastiCache Redis: ~$25/mo (cache.t4g.micro)
  • ALB: ~$20/mo
  • CloudFront: ~$5/mo (free tier covers most)
  • S3: ~$2/mo
  • ECR: ~$1/mo
  • VPC, Route53, CloudWatch: ~$10/mo
  • Total: ~$118/mo for dev environment

This is the cost of not cutting corners. It's the cost of sleeping well at night.
